---

## Purpose

This system generates complete, high-quality HTML lesson plans for the Kenya CBE curriculum at scale. It targets Grades 10–13 STEM subjects (Physics, Biology, Chemistry, Mathematics) and produces 2,000+ lessons following the Kenya CBE phenomenon-driven format.

---

## What "CBE Format" Means

Each generated lesson plan follows Kenya's Competency-Based Education structure:

- **Section A**: Specific Learning Outcomes (SLOs) and overview with Key Inquiry Questions
- **Section B**: 5-Column Implementation Framework (Learning Experience, Key Questions, Learning Resources, Assessment, Values/PCIs)
- **Section C**: Teacher Reflection Questions
- **Section D**: Summary Table Prompts
- **Section E**: Differentiation Strategies and Formative Assessments

Lessons embed Kenyan context, NGSS storyline methodology, and ARES resource placeholders. Raw AI output uses `===SECTION X:...===` delimiters that are post-processed into HTML.

## System Design Decisions

### Multi-Model Pipeline
Three models are used in sequence per lesson to balance cost and quality:
1. **Gemini 3 Flash** — fast, cheap draft generation (~$0.017/lesson)
2. **GPT-5** — structures draft into CBE template format (~$0.054/lesson)
3. **Claude Sonnet** — polishes top 50% of lessons by quality score (~$0.042/lesson for half)

Total: ~$0.123/lesson, $246 for 2,000 lessons (vs $1,200 for Claude-only).

### Two Execution Paths
1. **Full Spark pipeline** (`src/main_pipeline.py`): Production path with distributed processing, GPU embeddings, ChromaDB RAG, and Spark orchestration of API calls.
2. **Standalone generators** (`src/production_generator_complete.py`, `src/generate_with_ai.py`, etc.): Simpler scripts for single-subject or small-batch runs, calling Claude directly without Spark.

### RAG Context Injection
Before generation, the system performs semantic retrieval from ChromaDB (populated with curriculum PDF chunks) to inject relevant curriculum context into each generation prompt. This grounds lessons in the actual Kenya curriculum content.

## Curriculum Inputs

PDFs go in `data/raw/curriculum_pdfs/`. Expected naming convention:
```
{Subject}-Grade-{N}-June-2024.pdf
```
Example: `Physics-Grade-10-June-2024.pdf`

The `ResourceManager` (`src/resource_manager.py`) resolves PDFs via `config/generation_config.yaml` first, then falls back to filename pattern matching.

## Key Configuration Files

| File | Purpose |
|------|---------|
| `config/generation_config.yaml` | Active config for standalone generators (paths, curriculum sources, template examples) |
| `config/config.yaml` | Spark pipeline config (auto-generated by `setup.sh`) |
| `templates/cbe_lesson_template.md` | Canonical CBE lesson template injected into AI prompts |
| `templates/template_config.yaml` | Points `TemplateLoader` to the template file |
| `.env` | API keys (not committed) |

## Output

Generated lessons land in `data/outputs/html/` as individual HTML files named `Lesson_{N:04d}_lesson_{N:04d}.html`. Each is MS Word-compatible and 3,000–5,000 words. Validation pass rate targets 90%+.
